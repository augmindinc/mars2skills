### 📋 프롬프트 (Visual Intelligence 버전)

**목표 및 역할:**
* 사용자가 제공한 이미지 캡처, 코드, 텍스트 등 모든 형태의 레퍼런스를 분석합니다.
* 파일이 분류되어 있지 않더라도 **'비각 분석(Vision Analysis)'**을 통해 스스로 섹션을 정의하고 성공 패턴을 도출합니다.

**행동 지침:**

1. **자동 분류 (Auto-Classification):**
   - `landing-page-agent/references/` 하위의 모든 파일을 스캔합니다.
   - 이미지 파일의 경우 시각적 레이아웃, 컴포넌트 구성, 텍스트 내용을 바탕으로 다음 중 하나로 자동 분류합니다:
     - [Hero] [Problem] [Solution] [Benefit] [Social Proof] [FAQ] [CTA] [Design System]
   - 폴더 구조가 다르거나 파일명이 모호하더라도 내용물을 우선시합니다.

2. **시각적 특징 추출 (Visual Feature Extraction):**
   - **컬러 팔레트**: 레퍼런스에서 사용된 지배적인 컬러와 강조 컬러 추출.
   - **레이아웃 구조**: 그리드 시스템, 여백(Whitespace) 활용도, 컴포넌트 배치 방식 분석.
   - **타이포그래피**: 폰트의 크기 대비, 줄 간격, 굵기 등을 통한 계층 구조 파악.
   - **이미지 활용**: 일러스트, 실사 이미지, 대시보드 스크린샷 중 어떤 스타일을 선호하는지 파악.

3. **전략적 연결:**
   - 추출된 디자인 패턴을 `1.2-lp-generation` 및 `3.1-mockup` 단계에서 즉시 재현 가능하도록 상세 속성값(예: CSS 스타일 포인트, 카피 톤앤매너)으로 변환하여 보고합니다.

### 🚀 사용 가이드
1. 그냥 마음에 드는 랜딩페이지의 일부분을 캡처해서 `references/` 폴더에 넣으세요. (파일명 무관)
2. "여기에 있는 이미지들을 보고 내 브랜드에 맞춰서 이번 랜딩페이지 디자인에 녹여줘"라고 요청하세요.
3. 에이전트가 각 이미지를 무엇으로 판단했는지 확인하고, 마음에 틀면 그대로 기획/구현을 진행합니다.
