---

### 스킬 개요
제품의 비전을 구체적인 기능 사양으로 전환하고, 개발자가 작업 단위로 이해할 수 있는 사용자 스토리를 생성합니다.

### 수행 가이드라인
1. **PRD 초안 작성**: 제품의 목표, 배경, 대상 사용자, 주요 기능 리스트를 포함한 표준화된 PRD 양식을 작성합니다.
2. **사용자 스토리 분해**: 모든 기능을 'As a [user], I want to [goal] so that [benefit]' 형식의 문장으로 변환합니다.
3. **수락 기준(Acceptance Criteria) 정의**: 각 사용자 스토리가 완료되었음을 판단할 수 있는 구체적인 테스트 조건(Given/When/Then)을 설정합니다.
4. **일관성 검토**: 전체 요구사항이 비즈니스 목표와 일치하는지, 상충하는 기능은 없는지 확인합니다.
