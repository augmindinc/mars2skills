### 스킬 개요
비즈니스 요구사항을 기술적인 데이터 구조와 통신 규격으로 변환하여 개발팀과의 소통 비용을 줄입니다.

### 수행 가이드라인
1. **데이터 모델링**: 주요 엔티티(Entity) 간의 관계(Relationship)를 정의하고, 필요한 필드와 데이터 타입을 명시합니다. (ERD 초안 포함)
2. **API 경로 설계**: RESTful 원칙에 따라 명확하고 일관된 API 엔드포인트 경로를 정의합니다.
3. **요청/응답 명세**: 각 API의 Request Body와 Response Body 형식을 JSON 등으로 구체화합니다.
4. **OpenAPI 준수**: 서비스 확장을 위해 OpenAPI(Swagger) 표준 규격을 최대한 준수하여 문서를 작성합니다.
